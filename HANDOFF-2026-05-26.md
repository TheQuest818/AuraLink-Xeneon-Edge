## What this project is

AuraLink is a Python desktop overlay (pywebview + WebView2) that runs full-screen
on a secondary Corsair Xeneon Edge display (2560×720). It renders a Sonar audio
mixer + Spotify now-playing + stream-deck-style buttons + utility panel.

**Stack:** `main_v5.py` (Python backend, audio + Spotify + Discord + GSMTC),
`index_v5.html`, `app_v5.js`, `style_v5.css` (WebView2 frontend).
Older v3 / v4 files exist as reference but are not loaded.

**Launch:** `C:\Users\User\AppData\Local\Programs\Python\Python311\pythonw.exe "D:\AuraOverlay\AuraLinkv2\main_v5.py"`

**Cache-bust:** on launch, `main_v5.py` rewrites the HTML to add `?v=<timestamp>`
query strings on CSS/JS refs, so WebView2 always picks up fresh edits. **Any
JS/CSS/HTML change requires closing + relaunching AuraLink** — there is no hot
reload.

## What this session shipped

1. **Pause-on-game** — when Sonar's game channel has any session with
   `peak > 0`, the background video fades to a configurable pause image (or
   black). 5s debounce on resume to prevent flapping. Devtools helpers
   `window.setPauseImage()` / `window.clearPauseImage()`.
2. **TILES opacity slider** in the font picker modal (F button). Drives
   `--tile-alpha` CSS variable that scales background/borders/blur/shadow on
   `#app .glass-card` — content (text, sliders, color swatches) stays solid.
   Persisted as `localStorage.tileOpacity`.
3. **GSMTC pipeline for Spotify** — Python polls `GlobalSystemMediaTransportControlsSession`
   every 500ms, filters to `source_app_user_model_id` containing "spotify",
   pushes updates to JS via `evaluate_js("window._gsmtcUpdate(...)")`.
   Replaces the slow 30s Web API polling for local playback. Includes
   thumbnail extraction (base64 on track change only) and position
   interpolation using `timeline.last_updated_time`.
4. **Clickable shuffle / repeat** — the SHUFFLE and REPEAT spans in the Spotify
   widget now toggle via GSMTC's `try_change_shuffle_active_async` /
   `try_change_auto_repeat_mode_async`. Optimistic UI with rollback on error.
5. **Diagnostic infrastructure** — `_debug_log(msg)` at module level writes
   to `sonar_debug.log` from any thread. GAME-CH change events logged when the
   process set on Sonar game channel changes. `debug=True` on
   `webview.start()` enables right-click → Inspect.

## Conventions to follow

- **Python edits go in `main_v5.py`**, JS in `app_v5.js`, CSS in `style_v5.css`,
  HTML in `index_v5.html`. Don't touch the v3 / v4 files.
- **JS↔Python bridge** is `window.pywebview.api.<method>()`. Add new methods to
  the `SonarAPI` class. Python returns JSON-serializable dicts.
- **Background threads** that need to push to JS should use
  `window.evaluate_js("...")` via a global window ref (see `_gsmtc_window_ref`
  pattern). Pywebview marshals to the JS thread internally.
- **Logging** — use `_debug_log()` (module-level) from background threads;
  `_log()` (closure inside `main()`) from inside `main()`. Both write to
  `sonar_debug.log`. There is a known thread-safety issue (see Open Bugs).
- **localStorage keys in use:** `bgImage`, `bgVideo`, `bgPauseImage`,
  `channelFonts`, `tileOpacity`, `channelColors`, `channelOrder`,
  `streamDeckButtons`, `utilityBarButtons`, `channelFonts`. Don't collide.
- **CSS:** `--tile-alpha` is the global tile-opacity multiplier (`:root`).
  `.glass-card` rules under `#app` use `calc(orig * var(--tile-alpha))`.
  Modals (`.picker-card.glass-card`) intentionally inherit base `.glass-card`
  styles unscaled.

## Open bugs from the last `/code-review` (severity-ranked)

Only #1 was fixed (the `card-enter` animation override). 13–14 remain. None
urgent, all real. If the user asks for any of these, you have full context:

1. ~~`.glass-card` card-enter animation overrode inline opacity~~ — **FIXED**
2. `applyTileOpacity` (`#app .glass-card` scope) — the `.picker-card.glass-card`
   modal lives OUTSIDE `#app` so it's already excluded; verify when adding new
   modals.
3. `clearBgImage` (long-press BG) doesn't unmount `#bg-pause-image` or clear
   `localStorage.bgPauseImage`. Leaves stale state.
4. `pick_bg_image` wipes every file in `bg/` (including `pause_image.*`) but
   JS-side `bgPauseImage` URL keeps pointing at the deleted file → 404 next
   time pause overlay activates.
5. `applyBgImage` (static-image bg branch) doesn't strip `body.game-paused`
   class → if user switches from video to image while paused, the pause
   overlay stays on top of the new image.
6. `restoreBgImage` wraps everything in one try/catch — a corrupted `bgVideo`
   JSON kills the `bgImage` AND `bgPauseImage` restoration too.
7. `pauseForGame()` returns early when no `#bg-video` element exists, so the
   pause-image overlay never activates for image-only backgrounds.
8. `_audio_poll_loop` captures `sonar_device_ids` ONCE at thread start. If
   Sonar isn't running yet when AuraLink launches, the loop tracks 0 channels
   forever. No re-enumeration.
9. `pick_bg_image` can't delete the currently-playing video file (locked by
   WebView2) — `os.remove` silently swallows the PermissionError, leaves
   zombie files of different extensions in `bg/`.
10. `fetchLevels` silently swallows errors with `catch (_)` — transient
    `get_levels()` failure can leave `_auraGameActive` stale, premature resume.
11. `activeFontZone` never persisted to localStorage — always resets to
    'headers' on reload. Re-clicking a font writes to wrong zone.
12. `pauseForGame` setTimeout(700ms) closes over the video element reference
    at scheduling time. If video is replaced mid-pause, timer fires on stale
    element.
13. `clear_pause_image` uses `sys._MEIPASS` for frozen builds — same
    PyInstaller onefile breakage as `pick_pause_image` / `pick_bg_image`.
    (User runs from `.py` directly, so not currently hit.)
14. TILES slider only listens for `input` event, no `change` fallback.
15. `_debug_log` / `_log` / `js_log` write `sonar_debug.log` from multiple
    threads without a lock — bytes can interleave on Windows under
    concurrent writes.

## Current state at end of session

- GSMTC poll loop runs every 500ms; pushes only on state change (title, artist,
  album, playing, shuffle, repeat, art, duration) or backward-seek > 1.5s.
